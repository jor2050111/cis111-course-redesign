# Module 5: Software Liability and Secure Coding

Software powers critical systems, from healthcare devices to financial platforms to autonomous vehicles. When software fails, the consequences can range from minor inconvenience to loss of life. Who is responsible when a software defect causes harm? What ethical obligations do developers and organizations have to write secure, reliable code?

This module examines software liability frameworks, the ethics of secure coding practices, and how cost/benefit analysis applies to IT decision-making. You will evaluate real-world cases where software failures had significant consequences and analyze the ethical responsibilities at each stage of the software development lifecycle.

## Module Overview 🧭

* **Estimated time:** 4-5 hours (reading + activities + discussion)
* **Prerequisites:** Modules 1-4
* **Deliverables:** Discussion post, Quick Checks, Skills Lab analysis
* **CLOs addressed:** CLO VIII, CLO IX

## Learning Objectives 🎯

By the end of this module, you will be able to:

* **5.1-LO1 (Analyze):** Analyze the legal and ethical dimensions of software liability when software defects cause harm (CLO VIII)
* **5.2-LO2 (Apply):** Apply cost/benefit analysis to evaluate ethical trade-offs in IT development and deployment decisions (CLO IX)
* **5.3-LO3 (Evaluate):** Evaluate the ethical obligations of developers and organizations to implement secure coding practices (CLO VIII)
